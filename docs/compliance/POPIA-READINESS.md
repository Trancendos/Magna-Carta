# POPIA Readiness Annex (South Africa)

**Version:** 1.0.0  
**Date:** 2026-06-07  
**Owner:** DPO  
**Legislation:** Protection of Personal Information Act 4 of 2013 (POPIA)  
**Status:** ❌ Not applicable today — ✅ Programme future-readiness

---

## 1. Trigger

Activate when processing personal information of data subjects in South Africa or establishing ZA operations.

---

## 2. POPIA ↔ control mapping

| POPIA condition | Magna Carta artefact |
|-----------------|----------------------|
| Accountability (s.8) | ISMS, RACI, COMPLIANCE-BLUEPRINT |
| Processing limitation (s.9–12) | ROPA, purpose binding |
| Security safeguards (s.19) | POL-SEC-001, encryption |
| Data subject participation (s.23–25) | PROC-DSR-001 |
| Operator agreements (s.21) | POL-SUP-001, DPA templates |
| Cross-border transfer (s.72) | SCCs, transfer assessment |

---

## 3. Activation checklist

- [ ] Information Officer registration (if required)
- [ ] PAIA manual if public body customer (unlikely)
- [ ] Operator agreements with ZA subprocessors
- [ ] ROPA extension for ZA data subjects
- [ ] Security compromise notification procedure (align PROC-IR-001)
